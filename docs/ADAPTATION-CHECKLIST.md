# Adaptation Checklist

Use this when turning Noir Bistro into a client-specific version.

## Brand changes

1. Replace the demo name and cuisine positioning
2. Update the hero headline and intro paragraph
3. Swap signature dishes and wine references
4. Adjust the color mood if the client brand is lighter or more modern

## Content changes

1. Replace the story section with the real brand narrative
2. Update menu cards with actual dishes
3. Change private dining content to events, rooftop service, chef collaborations, or hotel dining if needed
4. Update reservation language to match the client's actual booking process

## Visual changes

1. Swap the typography only if the client already has a strong brand font direction
2. Adjust the hero mood toward steakhouse, omakase, Mediterranean, or hotel lounge if needed
3. Keep the structure unless there is a strong reason to change it

## Technical changes

1. Keep the performance-aware behavior
2. Replace the demo form feedback with a real booking action later
3. Move to a framework stack only when the client scope requires multiple pages or integrations
